A partir de spec.md y plan.md, genera tasks.md con tareas concretas para implementar la web en GitHub Pages.
Organiza las tareas por fases.
El archivo debe ser claro y conciso, y NO debe incluir ninguna línea de código.
Usa checkboxes Markdown (- [ ]) para cada tarea.
Cada task debe incluir: descripción corta, archivos/carpetas sugeridas, dependencias y definición de “done”.
